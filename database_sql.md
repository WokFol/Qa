# 📊 Работа с базами данных (SQL)

В этом разделе собраны примеры SQL-запросов, для поиска проверки и тестирования базы данных

### 🔎 1. Поиск конкретного пользователя (Фильтрация)
Используется для проверки, что новый пользователь успешно записался в базу данных после отправки POST-запроса.
```sql
SELECT id, username, email, is_active 
FROM users 
WHERE email = 'test_user@example.com';
```

### 🔗 2. Проверка заказов пользователя (Объединение таблиц JOIN)
Используется, когда нужно связать данные из таблицы пользователей и таблицы их покупок, чтобы проверить историю заказов.
```sql
SELECT users.username, orders.order_id, orders.amount, orders.status
FROM orders
INNER JOIN users ON orders.user_id = users.id
WHERE users.id = 45;
```

### 📈 3. Подсчёт количества покупок (Группировка данных)
Используется для аналитических проверок (например, найти пользователей, у которых больше 5 заказов).
```sql
SELECT user_id, COUNT(order_id) AS total_orders
FROM orders
GROUP BY user_id
HAVING COUNT(order_id) > 5;
```

### ❌ 4. Поиск багов и аномалий в данных (Тестирование бизнес-логики)
**Сценарий:** Проверка на логическую ошибку в БД. В интернет-магазине статус заказа не может быть «Доставлен» (`delivered`), если у него отсутствует дата доставки или он числится как «Не оплачен». Этот запрос помогает QA мгновенно найти такие баги в данных.

```sql
SELECT order_id, user_id, status, payment_status, delivery_date
FROM orders
WHERE status = 'delivered' AND (delivery_date IS NULL OR payment_status = 'unpaid');
```
*Если запрос вернет хотя бы одну строку — значит, в системе есть критический баг оформления статусов.*

### ⚠️ 5. Проверка граничных значений и валидации (Тестирование цен)
**Сценарий:** Проверка безопасности и валидации полей. Цены товаров или суммы заказов в базе данных никогда не должны быть отрицательными или равными нулю. QA использует этот запрос для поиска товаров, у которых цена из-за бага ушла в минус.

```sql
SELECT product_id, product_name, price, stock_quantity
FROM products
WHERE price <= 0 OR stock_quantity < 0;
```
*Этот запрос помогает убедиться, что бэкенд и база данных правильно валидируют финансовые поля и не пропускают некорректные значения.*
