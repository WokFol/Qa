# Тестирование сайта Т-банк
##**Важно**

В данном проекте представлены примеры тест-кейсов и баг-репортов. Проект подготовлен ради ознакомления и не является официальной документацией Т-Банка.
## Тест кейсы

### Позитивные тест-кейсы

#### Тест кейс 1:Успешный вход в личный кабинет действующего клиента.

* **Предусловие:** Номер телефона зарегистрирован в системе. 
* **Шаги:** 
1. Открыть страницу входа в личный кабинет Т-Банка.
2. Ввести валидный номер телефона клиента. 
3. Нажать кнопку **«Войти»**. 
4.Ввести корректный код из СМС / Push-уведомления.
* **Ожидаемый результат:** Пользователь успешно авторизован, открывается главная
страница личного кабинета. 

### Негативные тест кейсы

#### Тест кейс 2: Отправка пустой формы авторизации.

* **Шаги:**
1. Оставить поле «Номер телефона» пустым. 
2. Нажать кнопку **«Получить код»**. 
* **Ожидаемый результат:** Кнопка неактивна или появляется сообщение об ошибке: 
*«Поле обязательно для заполнения»*. Запрос на бэкенд не отправляется. 

#### Тест кейс 3: Попытка входа с вводом невалидного СМС-кода.

* **Шаги:** 
1. Ввести валидный номер телефона. 
2. Нажать кнопку **«Получить код»**. 
3. Ввести неверный код из СМС (например, `0000`). 
* **Ожидаемый результат:** Система выдает ошибку: *«Неверный код. Попробуйте ещераз»*. 


#### Тест кейсы 4: Ввод слишком короткого номера телефона (граничные значения).

* **Шаги:**
1. Ввести неполный номер телефона (например, всего 5 цифр вместо 10). 
2. Попробовать нажать кнопку **«Получить код»**. 
* **Ожидаемый результат:** Кнопка «Получить код» остается заблокированной,либо система сразу выдает ошибку: *«Номер телефона введен не полностью»*. 

#### Тест кейсы 5: Попытка ввода букв и спецсимволов (защита поля ввода).
* **Шаги:**
1. Попробовать ввести в поле номера телефона текст `Иван%#@`. 
* **Ожидаемый результат:** В поле невозможно ввести ничего, кроме цифр. Буквы и
спецсимволы автоматически игнорируются фильтром (маской) ввода. 


## Баг-репорты
	
### 🔴 Баг-репорт 1: Маскировка пароля/кода в поле ввода СМС.

* **Компонент:** Форма ввода СМС-кода авторизации.
* **Серьезность (Severity):** High (Высокая). 
* **Шаги для воспроизведения:** 
1. Ввести валидный номер телефона и нажать **«Войти»**. 
2. Дождаться СМС с кодом и начать вводить цифры в поле. 
* **Фактический результат:** Вводимые цифры кода отображаются на экране в открытом виде и не скрываются точками. 
* **Ожидаемый результат:** Символы СМС-кода должны маскироваться (превращаться в точки) сразу после ввода для безопасности. 

### 🟡 Баг-репорт 2: Бесконечный таймер повторной отправки СМС. 

* **Компонент:** Модуль отправки СМС-кода. 
* **Серьезность (Severity):** Medium (Средняя). 
* **Шаги для воспроизведения:** 
1. Ввести валидный номер телефона и нажать **«Получить код»**.
2. На этапе отсчета таймера отключить интернет на устройстве. 
3. Дождаться завершения времени таймера (`00:00`) и включить интернет. 
* **Фактический результат:** Таймер застывает на значении `00:00`. Кнопка «Отправить код повторно» остается неактивной. * **Ожидаемый результат:** После дохождения таймера до `00:00` кнопка «Отправить код повторно» становится активной для клика.
